# 人民法院案例库案例编写参考指南

## 一、案例库稿件与裁判文书的区别

| 维度 | 裁判文书 | 案例库稿件 |
|------|---------|-----------|
| 目的 | 解决具体争议 | 提炼裁判规则供类案参考 |
| 视角 | 针对个案当事人 | 面向司法实务全体 |
| 详略 | 全面记录程序和实体 | 聚焦裁判理由和规则 |
| 语言 | 法院裁判语言 | 精炼的规则陈述语言 |
| 结构 | 按文书格式规范 | 按案例库编写规范 |

## 二、各模块详细编写规范

### 2.1 标题编写规范

**格式**：`[当事人A]与[当事人B][案由]一案`

**规则**：
- 当事人A为原告/上诉人/申请人，当事人B为被告/被上诉人/被申请人
- 多个当事人时，取主要当事人，其余用"等"概括
- 当事人为自然人的，使用"姓+某某"格式
- 当事人为法人/非法人组织的，使用完整名称
- 案由使用《民事案件案由规定》《刑事案件罪名规定》等标准案由

**示例**：
- ✅ `张某某与某科技有限公司房屋租赁合同纠纷一案`
- ✅ `李某某等与王某某民间借贷纠纷一案`
- ❌ `张某诉某公司租房案`（案由不规范、当事人未脱敏）
- ❌ `关于李四借款的判决`（格式错误）

### 2.2 副标题（法律问题）编写规范

**格式**：`——[核心法律问题的疑问句或陈述句]`

**规则**：
- 从案件中提炼出一个核心法律问题
- 问题应具有普遍性，非个案特有问题
- 优先使用疑问句形式

**示例**：
- ✅ `——未签订书面合同但实际履行主要义务的，合同是否成立？`
- ✅ `——用人单位以"客观情况发生重大变化"解除劳动合同时的审查标准`
- ❌ `——张某某能不能要回房子`（过于口语化、过于具体）

### 2.3 关键词编写规范

**数量**：3-6 个，以分号分隔

**来源**：
1. 案由关键词（如"房屋租赁合同纠纷"→"租赁合同"）
2. 法律制度关键词（如"合同成立""实际履行"）
3. 裁判规则关键词（如"口头合同效力""主要义务认定"）

**排序**：按重要性从高到低排列

**示例**：`合同成立；实际履行；口头合同；合同形式要件`

### 2.4 基本案情编写规范

**结构模板**：

```
[当事人A]（脱敏后简介）。
[当事人B]（脱敏后简介）。
[A的诉讼请求/申请请求]。
[B的抗辩/答辩意见]。
[法院查明的事实，按时间顺序陈述]。
```

**编写要点**：
1. 事实陈述只记录"是什么"，不记录"认为什么"
2. 与裁判理由无直接关联的程序性事实可省略
3. 金额保留但脱敏银行账号等具体信息
4. 时间用"YYYY年MM月DD日"格式
5. 不引用原文书段落编号

**示例**：

> 20XX年X月，张某某与某科技有限公司签订房屋租赁合同，约定张某某承租某市某区XX路XX号房屋用于经营餐饮，租赁期限三年，月租金人民币X万元。
>
> 合同履行过程中，因张某某未按期支付第三季度租金，某科技有限公司于20XX年X月X日向法院提起诉讼，请求解除租赁合同并要求张某某支付欠付租金及违约金。张某某抗辩称，其迟延支付系因公司经营困难，且已在诉前补交了租金，不同意解除合同。
>
> 法院查明：张某某于20XX年X月X日将欠付租金转入某科技有限公司账户，此时距约定付款日已逾期XX天。双方合同约定逾期付款超过XX天出租方有权解除合同。

### 2.5 裁判理由编写规范

**结构模板**：

```
本案争议焦点为：[争议焦点1]；[争议焦点2]。

关于[争议焦点1]。[当事人A主张]。[当事人B抗辩]。法院认为，[法律分析过程]。[法律依据]。[结论]。

关于[争议焦点2]。……
```

**编写要点**：
1. 争议焦点应精炼为一句问句或判断句
2. 每个争议焦点独立成段分析
3. 分析顺序：当事人主张 → 法律依据 → 法院认定 → 结论
4. 法条引用格式：《法律名称》第X条第X款
5. 保持法院原文说理逻辑，不添加学理分析

### 2.6 裁判要旨编写规范

**这是案例库稿件的核心部分，分两个层次：**

#### 裁判规则（1-3条）

将法院对具体个案的裁判，抽象为可适用于同类案件的一般性规则。

**提炼方法**：
1. 提取裁判文书"本院认为"部分的核心法律适用逻辑
2. 去除个案特有事实，保留法律适用的条件要素
3. 用规范的法律语言表述

**格式模板**：
- "[条件/情形描述]的，[法律适用结论]。"
- "[主体]在[情形]下，[权利义务/法律后果]。"

**示例**：
- 原文："本案中，张某某虽迟延支付租金，但在出租人行使解除权前已补交完毕，且迟延天数较短，未构成根本违约。"
- 提炼："承租人迟延支付租金但在出租人行使合同解除权前已补交，且迟延时间较短未致使合同目的不能实现的，不构成根本违约，出租人不得据此解除合同。"

#### 裁判要旨

对裁判规则的简要说明，解释规则的适用背景和界限。

**示例**：
> 上述裁判规则明确了迟延履行与合同解除之间的关系界限。在判断迟延履行是否构成根本违约时，应综合考量迟延时间、迟延金额占合同总额比例、是否已实际补正、守约方是否遭受不可弥补损失等因素。该规则对于规范租赁合同等继续性合同的解除权行使具有参考意义。

### 2.7 关联索引编写规范

**法律规范**：
- 列出本案适用的全部法律、法规、司法解释
- 格式：`《法律名称》第X条第X款` — 条文内容摘要
- 按效力层级排列：法律 → 行政法规 → 司法解释 → 地方性法规

**关联案例**（如有）：
- 与本案裁判规则相关或冲突的其他案例
- 格式：`案例名称（案号）` — 关联说明

**指导性案例/参考案例**（如有）：
- 最高人民法院发布的与本案相关的指导性案例
- 格式：`指导性案例第X号：案例名称` — 关联说明

### 2.8 人工复核清单规范

清单必须包含以下全部检查项：

1. **脱敏检查**：逐项确认所有自然人姓名、身份证号、住址、联系方式、银行账号等已脱敏
2. **事实核对**：将基本案情与原文书逐段比对，确认无事实遗漏或错误
3. **法条核验**：逐一核验法条编号和内容，确认现行有效
4. **案号核验**：确认案号格式完整、年份和序号正确
5. **裁判规则**：确认提炼的规则忠实于法院裁判逻辑，无过度抽象或不当归纳
6. **价值评估**：确认案例价值等级标注合理
7. **语言风格**：确认全文客观中立，无主观评价性语言
8. **关联索引**：确认法律规范列举完整，关联案例标注准确


## 四、常见错误与规避

| 错误类型 | 表现 | 规避方法 |
|---------|------|---------|
| 律师视角 | 从当事人角度分析胜败原因 | 全程使用法院视角，聚焦裁判逻辑 |
| 学术评论 | 对法条进行学理批判或修改建议 | 忠实呈现法院说理，不加学术评价 |
| 简单缩写 | 仅缩写文书内容，无规则提炼 | 必须完成裁判规则抽象和要旨提炼 |
| 过度抽象 | 规则提炼过于宽泛，脱离个案 | 规则应与个案事实保持逻辑关联 |
| 事实遗漏 | 省略影响裁判结论的关键事实 | 基本案情必须完整支撑裁判理由 |
| 法条错误 | 法条编号或内容与原文不符 | 逐条核验，使用法条检索工具验证 |
| 脱敏不彻底 | 保留可识别个人身份的信息 | 逐项检查所有个人信息字段 |
| 主观评价 | 使用"合理""正确""值得肯定"等评价词 | 使用中性陈述，不加价值判断 |
